## Change summary

> Enforces JVM target consistency across all subprojects (plugins) in `apps/mobile/android/build.gradle.kts`. Uses a dynamic Gradle `Provider` to sync `KotlinCompile` tasks with each project's `JavaCompile` task `targetCompatibility`. Resolves build failures caused by plugins like `receive_sharing_intent` defaulting to Java 1.8 while Kotlin tasks target a newer JVM.
