## Change summary

> `ProtectedRoute` conflated a failed `/auth/sync` (`role === null`) with "not admin" and redirected to `/no-access`, which has no gate and no role recheck — so a user who signed in while the API was down (or any transient sync failure) got stuck on `/no-access` even after the role hydrated correctly on a later refresh. New `SyncErrorScreen` (Retry + Sign out) renders when `firebaseUser` exists but role is `null`.
