## Change summary

> New `CrashReporter` helper (`apps/mobile/lib/core/observability/crash_reporter.dart`) that no-ops when Firebase isn't initialised and routes through `developer.log` in debug, so call sites stay safe everywhere. `initializeFirebase()` now disables Crashlytics collection in debug via `setCrashlyticsCollectionEnabled(!kDebugMode)` to keep local crashes out of the prod dashboard. `main.dart` seeds + listens to `authStateProvider` so every Crashlytics report carries the current Firebase uid. `SmsScanRepository` reports the previously-silent `/check` network failure as a non-fatal so outages stop hi
