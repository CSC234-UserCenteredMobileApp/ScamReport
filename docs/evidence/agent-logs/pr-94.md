## Change summary

> Flutter web (`flutter run -d chrome`) opens on a random localhost port that was not in `WEB_ORIGINS`, so the API blocked every preflight and the app could not talk to the backend. Added `^http://localhost:\d+$` and `^http://127.0.0.1:\d+$` regex entries to `WEB_ORIGINS` so any local dev tool reaches the API on `localhost:3000`. Production protection unchanged: a real cross-site attacker cannot forge `Origin: http://localhost` from a victim browser.
