## Change summary

> `requireRole` middleware previously read the caller's role from a Firebase custom claim, but **nothing in the repo ever calls `setCustomUserClaims`** — every real ID token decoded with `role == undefined`, so admin routes (`GET /admin/reports/queue`, the moderation flow, etc.) returned 403 for every user including real admins. Switch the lookup to Postgres `users.role` via Prisma, matching the source of truth declared in DATABASE_DESIGN §4.1 and PRD §2 ("Promoted manually").
